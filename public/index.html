Using the REST API

The Online Shop API allows you to perform operations related to e-commerce functionalities. You can manage users, products, and categories through various endpoints. Download the Postman Collection above to explore and test the API endpoints interactively.

Authentication

Authentication is handled via JSON Web Tokens (JWT). To authenticate your requests, use the /users/login endpoint with valid credentials to obtain a token. Include this token in the Authorization header for subsequent requests that require authentication.

Endpoints

Some of the available endpoints include:

  • POST /users/register: Register a new user.
  • POST /users/login: Authenticate and retrieve a JWT token.
  • POST /products: Create a new product (Admin only).

Using GraphQL

In addition to the REST API, you can also interact with the server using GraphQL. GraphQL allows you to request specific data and aggregate responses, which can reduce the number of requests and optimize the data transmission.

GraphQL Playground

Open the GraphQL Playground using the link above. The playground allows you to write, test, and optimize your GraphQL queries before implementing them in your application.

Queries and Mutations

In GraphQL, you retrieve data using Queries and modify data using Mutations. Here are some examples of how you might interact with the API using GraphQL:

Query Example: Fetch All Categories

query {
  categories {
    id
    name
  }
}

Mutation Example: Create a New Product

mutation {
  createProduct(createProductData: {categoryId: 1, name: "New Product", price: 99}) {
    id
    name
    price
  }
}
